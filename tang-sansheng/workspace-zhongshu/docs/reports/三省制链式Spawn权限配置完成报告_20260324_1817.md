## ✅ 配置修改

### 修改前

**中书省**:
```json
{
  "id": "zhongshu",
  "name": "中书省",
  ...
  // ❌ 无subagents配置
}
```

**门下省**:
```json
{
  "id": "menxia",
  "name": "门下省",
  ...
  // ❌ 无subagents配置
}
```

**尚书省**:
```json
{
  "id": "shangshu",
  "name": "尚书省",

  }
}
```

---

### 修改后

**中书省**:
```json
{
  "id": "zhongshu",
  "name": "中书省",
  ...
  "subagents": {
    "allowAgents": ["menxia"]  // ✅ 可以spawn门下省
  }
}
```

**门下省**:
```json
{
  "id": "menxia",
  "name": "门下省",
  ...
  "subagents": {
    "allowAgents": ["shangshu"]  // ✅ 可以spawn尚书省
  }
}
```

**尚书省**:
```json
{
  "id": "shangshu",
  "name": "尚书省",
  ...
  "subagents": {
    "allowAgents": ["zhongshu", "menxia", "yushitai", "bingbu", ...]  // ✅ 保持原有权限
  }
}
```

## ✅ 优势

### 1. 保持三省制分权制衡
- ✅ 中书省只能spawn门下省（不能跳过）
- ✅ 门下省只能spawn尚书省（不能跳过）
- ✅ 尚书省统一调度六部

### 2. 实现自动化
- ✅ 每个部门完成后自动触发下一个部门
- ✅ 陛下可以睡觉，AI自主完成项目
- ✅ 无需陛下手动触发每个环节

### 3. 流程清晰
- ✅ 中书省起草 → 门下省审核 → 尚书省派发 → 六部执行
- ✅ 每个环节职责明确
- ✅ 不会出现流程混乱

## 🎯 解决的问题

### 问题1：陛下睡觉系统停滞
- ❌ **修改前**: 陛下不在，中书省无法触发门下省
- ✅ **修改后**: 中书省可以自动spawn门下省

### 问题2：无法自动化执行
- ❌ **修改前**: 每个环节需要陛下手动触发
- ✅ **修改后**: 每个部门自动触发下一个部门

### 问题3：分权制衡被破坏
- ❌ **担心**: 给中书省spawn权限会破坏三省制
- ✅ **解决**: 链式spawn，每个部门只能spawn下一个部门
