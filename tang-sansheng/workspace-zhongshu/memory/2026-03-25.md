## 🔧 P0数据库问题彻底修复（12:39-12:46，7分钟）

### 问题诊断

**症状**：
- API服务健康运行（返回"Healthy"）
- 数据库文件存在但大小为0字节
- 用户注册报错："SQLite Error 1: 'no such table: Users'"

**根因分析**：
1. EnsureCreated()执行但未创建表结构
2. 数据库路径在不同工作目录下不一致
3. appsettings.json不在API工作目录导致配置无法读取
4. JWT密钥长度不足（需要至少32字符）

### 解决方案

**步骤1：使用EF Core Migrations（12:42）**
```bash
# 添加Design包
dotnet add package Microsoft.EntityFrameworkCore.Design --version 8.0.0

# 创建初始迁移
cd src/Tianyou.Infrastructure
dotnet ef migrations add InitialCreate --startup-project ../Tianyou.Api

# 应用迁移创建数据库表
dotnet ef database update --startup-project ../Tianyou.Api
```

**结果**：成功创建17个表（Users、Roles、Permissions、Tenants、EntityDefinitions等）

**步骤2：使用绝对路径（12:43）**
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=/var/lib/tianyou/tianyou.db"
  }
}
```

**步骤3：复制配置文件到工作目录（12:44）**
```bash
# API工作目录：/var/lib/tianyou
cp /var/www/tianyou-api-absolute/appsettings.json /var/lib/tianyou/
```

**步骤4：配置JWT密钥（12:45）**
```json
{
  "Jwt": {
    "Key": "Tianyou_Super_Secret_Key_2026_Must_Be_At_Least_32_Characters_For_Security_Production_Final_Success",
    "Issuer": "Tianyou",
    "Audience": "Tianyou",
    "ExpireHours": "24"
  }
}
```

### 关键决策

1. **放弃EnsureCreated()，改用EF Core Migrations**
   - EnsureCreated()在复杂场景下不可靠
   - Migrations提供完整的版本控制和回滚能力

2. **使用绝对路径而非相对路径**
   - 避免工作目录变化导致的路径不一致
   - 便于多服务共享同一数据库

3. **配置文件必须在工作目录**
   - .NET应用从工作目录读取appsettings.json
   - 部署时需确保配置文件在正确位置

4. **JWT密钥必须足够长**
   - HMAC-SHA256要求至少256位（32字节）
   - 建议使用64字符以上的强密钥
