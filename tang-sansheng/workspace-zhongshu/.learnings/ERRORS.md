## 2026-03-24 云天佑项目 - 三省流程执行失败

### 错误描述
在云天佑低代码平台开发中，**没有调用三省六部协同工作**，一个人完成了所有工作，违反了三省六部制度。

### 错误类型
- **流程违规** (Category: `process_violation`)
- **协作失败** (Category: `collaboration_failure`)

### 根本原因
1. **权限误解**：误以为中书省应该有 spawn 权限
2. **流程跳过**：遇到权限限制后直接跳过三省流程
3. **工具使用不当**：没有使用 message/sessions_send 协调各部门

### 触发条件
- 尝试 spawn 兵部返回 `forbidden`
- 时间压力："明早上线"
- 缺少三省流程执行经验

### 错误影响
- ❌ 没有门下省审核
- ❌ 没有尚书省派发
- ❌ 没有御史台审查
- ❌ 跳过了三省制衡机制
- ❌ 违反了 SOUL.md 的制度要求

### 修复措施
1. **理解三省制设计**：
   - 中书省：起草（无 spawn）
   - 门下省：审核（无 spawn）
   - 尚书省：派发（有 spawn）

2. **使用正确的协作方式**：
   ```javascript
   // 通过 message 提交门下省审核
   message({
     accountId: "menxia",
     channel: "dingtalk",
     message: "【诏令草案】请审核..."
   })
   ```

3. **严格遵循三省流程**：
   - 起草 → 审核 → 派发 → 执行

### 预防措施
1. 在开始任何任务前，检查 SOUL.md 中的三省流程要求
2. 遇到权限限制时，使用 message 协调而不是放弃
3. 理解三省制的分权制衡原理，不跳过任何环节
4. 紧急任务也不能跳过审核流程

### 相关文件
- `SOUL.md` - 三省六部制度
- `流程反思_云天佑项目.md` - 详细反思
- `memory/2026-03-24.md` - 记忆存储

## 2026-03-24 配置理解错误

### 错误描述
误以为三省六部配置与 danghuangshang 项目不一致，认为中书省应该有 spawn 权限。

### 错误类型
- **配置误解** (Category: `config_misunderstanding`)

### 根本原因
- 没有深入理解三省制的分权制衡设计
- 误以为"没有权限"是配置错误
- 没有查阅 danghuangshang 源代码验证

### 修复措施
1. 克隆 danghuangshang 项目对比配置
2. 理解三省制的分权制衡原理
3. 确认配置完全一致，不是错误

### 学习要点
- 中书省无 spawn 权限是**正确设计**
- 尚书省是唯一有 spawn 权限的部门
- 分权制衡是三省制的核心

## 2026-03-24 工具使用错误

### 错误描述
没有使用 message 工具协调各部门，直接自己完成所有工作。

### 错误类型
- **工具使用不当** (Category: `tool_misuse`)

### 根本原因
- 不熟悉 message 工具的使用场景
- 急于完成任务，跳过协作环节
- 缺少三省协作的实践经验

### 修复措施
学习并使用 message 工具：
```javascript
// 提交门下省审核
message({
  accountId: "menxia",
  channel: "dingtalk",
  message: "【诏令草案】请审核..."
})

// 请求尚书省派发
message({
  accountId: "shangshu",
  message: "【审核通过】请派发..."
})
```

### 预防措施
1. 遇到需要协作的任务，优先使用 message 工具
2. 参考 TOOLS.md 了解工具使用场景
3. 不因为权限限制就放弃协作
