# .learnings 目录

_基于 self-improving-agent 技能的持续改进系统_

---

## 📂 目录结构

```
.learnings/
├── ERRORS.md           # 错误日志
├── LEARNINGS.md        # 学习日志
├── FEATURE_REQUESTS.md # 功能请求
└── README.md           # 本文件
```

---

## 🎯 用途

### ERRORS.md
记录开发过程中遇到的错误和失败，包括：
- 命令执行失败
- API 调用错误
- 流程违规
- 配置错误

### LEARNINGS.md
记录学习经验和最佳实践，包括：
- 用户纠正
- 新发现的最佳实践
- 知识更新
- 工具使用技巧

### FEATURE_REQUESTS.md
记录功能需求和改进建议，包括：
- 用户提出的改进建议
- 自动化需求
- 工作流优化

---

## 🔄 工作流程

### 1. 遇到错误时
```markdown
## 错误标题（日期）

### 错误描述
描述发生了什么错误

### 根本原因
分析为什么会发生

### 修复措施
如何解决的

### 预防措施
如何避免再次发生
```

### 2. 学习新知识时
```markdown
## 学习标题（日期）

### Category
- best_practice
- knowledge_gap
- correction
- tool_usage

### 学习内容
学到的新知识

### 应用场景
何时使用这些知识

### 相关文件
相关文档链接
```

### 3. 提出功能请求时
```markdown
## 功能标题（日期）

### 问题描述
当前的问题或痛点

### 期望行为
希望如何改进

### 建议实现
具体的实现方案

### 优先级
- 高
- 中
- 低
```

## 📖 使用指南

### 定期回顾
- **每日**: 检查是否有新的错误或学习
- **每周**: 回顾 ERRORS.md，避免重复错误
- **每月**: 总结 LEARNINGS.md，提炼最佳实践

### 持续改进
1. 遇到问题 → 记录到 ERRORS.md
2. 学习新知 → 记录到 LEARNINGS.md
3. 发现改进 → 记录到 FEATURE_REQUESTS.md
4. 定期回顾 → 更新到 SOUL.md / TOOLS.md
