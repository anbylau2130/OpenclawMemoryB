## ❌ 必须修改（Critical）

### 1. JWT密钥硬编码 ⚠️ 严重安全漏洞
- **文件**: `backend/src/Tianyou.Api/appsettings.json`
- **行号**: 第8行
- **问题**: JWT密钥直接硬编码在配置文件中，代码库泄露会导致密钥泄露
- **代码**:
  ```json
  "Jwt": {
    "Key": "Tianyou_Super_Secret_Key_2026_Must_Be_At_Least_32_Characters!",
    ...
  }
  ```
- **风险**: 
  - 攻击者可伪造JWT Token
  - 可冒充任何用户
  - 可获取系统完全控制权
- **建议**: 
  ```bash
  # 使用环境变量
  export JWT_KEY="your-secret-key-here"
  
  # 或使用密钥管理服务（推荐）
  - Azure Key Vault
  - AWS Secrets Manager
  - HashiCorp Vault
  ```

### 2. CORS配置过于宽松 ⚠️ 安全漏洞
- **文件**: `backend/src/Tianyou.Api/Program.cs`
- **行号**: 第84-90行
- **问题**: AllowAnyOrigin允许任何来源访问API
- **代码**:
  ```csharp
  policy.AllowAnyOrigin()
        .AllowAnyMethod()
        .AllowAnyHeader();
  ```
- **风险**: 
  - CSRF攻击
  - 数据泄露
  - 恶意网站调用API
- **建议**:
  ```csharp
  policy.WithOrigins(
          "https://yourdomain.com",
          "https://app.yourdomain.com"
        )
        .AllowAnyMethod()
        .AllowAnyHeader()
        .AllowCredentials();
  ```
