# Ralph Loop 快速参考卡

> **一页速查** - Ralph Loop 核心要点

---

## 🎯 核心理念

```
while :; do cat PROMPT.md | claude-code; done
```

**本质**: Bash 循环 + 全新上下文 + 文件持久化

---

## ⚡ 核心原则

### 1. One Item Per Loop
```
✅ 每次循环只做一件事
✅ 任务大小: 一个上下文窗口能完成
❌ 避免: "构建整个系统"
```

### 2. Fresh Context
```
每次迭代 = 新 AI 实例
记忆通过文件持久化:
  - Git history
  - progress.txt
  - prd.json
  - AGENTS.md
```

### 3. Trust Ralph
```
让 Ralph 决定优先级
LLM 擅长推理重要性
完全放手（hands-off）
```

### 4. Feedback Loops
```
必须有:
  ✅ 类型检查
  ✅ 单元测试
  ✅ 集成测试
  ✅ CI 保持绿色
```

## 🔄 工作流程

```
1. 创建 PRD
   → tasks/prd-[feature].md

2. 转换为 JSON
   → prd.json

3. 运行 Ralph
   → ./scripts/ralph/ralph.sh

4. 循环执行
   ┌─────────────────────┐
   │ 选择任务 (passes: false) │
   └──────────┬──────────┘
              ↓
   ┌─────────────────────┐
   │ 实现任务            │
   └──────────┬──────────┘
              ↓
   ┌─────────────────────┐
   │ 质量检查            │
   └──────────┬──────────┘
              ↓
   ┌─────────────────────┐
   │ 通过? → Commit      │
   │ 失败? → Fix         │
   └──────────┬──────────┘
              ↓
   ┌─────────────────────┐
   │ 更新 prd.json       │
   │ 追加 progress.txt   │
   └──────────┬──────────┘
              ↓
   ┌─────────────────────┐
   │ 所有完成? → EXIT    │
   │ 未完成? → Loop      │
   └─────────────────────┘
```

## ✅ 任务大小参考

### ✅ 正确的大小
```
- 添加数据库列和迁移
- 添加 UI 组件到现有页面
- 更新服务器操作逻辑
- 添加筛选下拉框
- 编写单元测试
```

### ❌ 过大的任务（需要拆分）
```
- "构建整个仪表板"
- "添加认证系统"
- "重构整个 API"
- "实现所有 CRUD 操作"
```

## 🛠️ 快速开始

### 安装
```bash
# 复制到项目
mkdir -p scripts/ralph
cp ralph.sh scripts/ralph/
cp prompt.md scripts/ralph/
chmod +x scripts/ralph/ralph.sh
```

### 运行
```bash
# 使用 Amp（默认）
./scripts/ralph/ralph.sh [max_iterations]

# 使用 Claude Code
./scripts/ralph/ralph.sh --tool claude [max_iterations]
```

### 监控
```bash
# 查看完成状态
cat prd.json | jq '.userStories[] | {id, title, passes}'

# 查看学习记录
cat progress.txt

# 查看 git 历史
git log --oneline -10
```

## 📝 AGENTS.md 应该记录什么

```markdown
## 模式发现

- API 路由遵循 RESTful 规范

## 陷阱和注意事项
- 不要忘记修改 Z 时更新 W
- 组件 X 和 Y 有循环依赖

## 有用的上下文
- 设置面板在 component X 中
- 测试数据库使用 SQLite
```

## 🚫 常见陷阱

| 陷阱 | 解决方案 |
|-----|---------|
| **任务过大** | 拆分为小任务 |
| **缺乏测试** | 建立反馈循环 |
| **不记录学习** | 更新 AGENTS.md |
| **过度干预** | 信任 Ralph |
| **上下文浪费** | 只加载必要信息 |

---

## 💡 最佳实践

1. **规范优先** - 先写规范，再实现
2. **持续调优** - 观察和改进 prompt
3. **上下文管理** - 珍惜每个 token
4. **最终一致性** - 保持信念和耐心
5. **自动优先级** - 让 Ralph 决定
