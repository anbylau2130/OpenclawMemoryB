## 🚀 项目配置

### 步骤1: 安装 Ralph

#### 方法A: 复制到项目
```bash
# 从项目根目录
cd /path/to/your/project

# 创建目录
mkdir -p scripts/ralph

# 复制文件
cp /path/to/ralph/ralph.sh scripts/ralph/
cp /path/to/ralph/prompt.md scripts/ralph/

# 添加执行权限
chmod +x scripts/ralph/ralph.sh
```

#### 方法B: 克隆仓库
```bash
git clone https://github.com/snarktank/ralph.git
cp -r ralph/scripts/ralph your-project/scripts/
```

#### 方法C: Claude Code Marketplace
```bash
# 添加 marketplace
/plugin marketplace add snarktank/ralph

# 安装 skills
/plugin install ralph-skills@ralph-marketplace
```

### 步骤2: 创建 PRD

#### 使用 PRD Skill
```bash
# 启动 AI 编码工具
amp
# 或
claude-code

# 加载 PRD skill 并创建
Load the prd skill and create a PRD for [你的功能描述]
```

#### 手动创建 PRD
```markdown
# tasks/prd-add-user-authentication.md

## 功能描述
为系统添加用户认证功能

## 用户故事

### Story 1: 登录页面
作为用户，我想要一个登录页面，以便访问系统

验收标准:
- 用户名和密码输入框
- 错误消息显示
- 登录成功跳转

### Story 2: 注册页面
作为新用户，我想要注册账号，以便使用系统

验收标准:
- 用户名、邮箱、密码输入
- 验证码发送
- 注册成功提示

### Story 3: JWT 认证
作为开发者，我想要 JWT 认证，以便保护 API

验收标准:
- 生成 JWT token
- 验证 JWT token
- 刷新 token

## 技术要求
- 使用 JWT 认证
- 密码加密存储
- 防止暴力破解
```

### 步骤4: 自定义 Prompt

#### 编辑 prompt.md
```markdown
# 项目配置

## 质量检查命令
- npm run typecheck
- npm run test
- npm run lint
- npm run build

## 代码库约定
- 使用 TypeScript strict 模式
- 组件使用 PascalCase 命名
- 函数使用 camelCase 命名
- 测试文件放在 __tests__/ 目录
- API 路由使用 RESTful 规范

## 常见陷阱
- 不要直接修改 dist/ 目录
- 环境变量在 .env.local 中配置
- API 路由需要认证中间件
- 数据库迁移需要先测试
- 不要跳过单元测试

## 有用的上下文
- 认证逻辑在 src/auth/ 目录
- API 路由在 src/api/ 目录
- 组件在 src/components/ 目录
- 工具函数在 src/utils/ 目录

## Ralph 指令
1. 选择最高优先级的未完成任务
2. 实现该任务（只实现这一个任务）
3. 运行质量检查
4. 如果检查通过，提交代码
5. 更新 prd.json 标记任务完成
6. 追加学习到 progress.txt
7. 更新 AGENTS.md（如果发现新模式或陷阱）
```

## 🏃 运行 Ralph

```bash
# 使用 Amp（默认）

# 使用 Claude Code
./scripts/ralph/ralph.sh --tool claude

./scripts/ralph/ralph.sh 20
```

### 监控运行

#### 查看任务状态
```bash

# 查看任务完成情况
cat prd.json | jq '.userStories[] | {id, title, passes}'

# 统计完成数量
cat prd.json | jq '[.userStories[] | select(.passes == true)] | length'
```

```bash
git log --grep="story-001" --oneline
```

## 🐛 故障排除

### 问题1: Ralph 一直失败某个任务

**可能原因**:
- 任务过大
- 缺少依赖
- 测试不通过

**解决方法**:
```bash
# 1. 拆分任务
将大任务拆分为多个小任务

# 2. 手动完成依赖
先手动完成依赖项

# 3. 修复测试
手动修复失败的测试

# 4. 更新 prompt.md
添加该任务的陷阱说明
```

### 问题2: Ralph 重复犯错

**可能原因**:
- 没有记录学习
- AGENTS.md 没有更新

**解决方法**:
```bash
# 1. 检查 progress.txt
cat progress.txt

# 2. 检查 AGENTS.md
cat AGENTS.md

# 3. 手动添加学习
echo "## 2026-03-23 学习\n- 不要使用 X 库，会导致 Y 问题" >> AGENTS.md

# 4. 更新 prompt.md
添加相关陷阱说明
```

### 问题3: 上下文溢出

**可能原因**:
- 规范文档太大
- 加载了不必要的文件

**解决方法**:
```bash
# 1. 精简规范文档
删除不必要的细节

# 2. 使用 Git 忽略
echo "large-file.json" >> .gitignore

# 3. 配置 Amp auto-handoff
# 在 ~/.config/amp/settings.json
{
  "amp.experimental.autoHandoff": { "context": 90 }
}
```

### 问题4: CI 一直失败

**可能原因**:
- 环境不一致
- 测试不稳定

**解决方法**:
```bash
# 1. 本地运行测试
npm run test

# 2. 检查 CI 配置
cat .github/workflows/ci.yml

# 3. 修复测试
# 手动修复不稳定的测试

# 4. 跳过测试（临时）
# 在 prompt.md 中临时禁用测试命令
```

## 📊 进阶技巧

### 技巧1: 多项目 Ralph

**场景**: 同时开发多个项目

**配置**:
```bash
# 项目1
project1/
├── scripts/ralph/
│   ├── ralph.sh
│   └── prompt.md
├── prd.json
└── progress.txt

# 项目2
project2/
├── scripts/ralph/
│   ├── ralph.sh
│   └── prompt.md
├── prd.json
└── progress.txt
```

**运行**:
```bash
# 终端1
cd project1 && ./scripts/ralph/ralph.sh

# 终端2
cd project2 && ./scripts/ralph/ralph.sh
```

### 技巧2: 共享 AGENTS.md

**场景**: 多个项目共享模式

**配置**:
```bash
# 创建共享目录
mkdir -p ~/shared-ralph

# 创建共享 AGENTS.md
cat > ~/shared-ralph/AGENTS.md << EOF
## 通用模式
- 使用 TypeScript strict 模式
- 函数必须有类型注解
- 测试覆盖率 > 80%

## 通用陷阱
- 不要使用 any 类型
- 不要跳过测试
- 不要忽略 lint 警告
EOF

# 在每个项目中创建符号链接
cd project1
ln -s ~/shared-ralph/AGENTS.md AGENTS.md
```

### 技巧4: Ralph Dashboard

**场景**: 可视化监控

**实现**:
```bash
# 创建简单监控脚本
cat > scripts/ralph-monitor.sh << 'EOF'
#!/bin/bash
while true; do
  clear
  echo "=== Ralph 监控 ==="
  echo ""
  echo "任务完成情况:"
  cat prd.json | jq '.userStories[] | "\(.id): \(if .passes then "✅" else "⏳" end) \(.title)"'
  echo ""
  echo "完成数量:"
  cat prd.json | jq '[.userStories[] | select(.passes == true)] | length'
  echo ""
  echo "最近学习:"
  tail -n 5 progress.txt
  echo ""
  sleep 10
done
EOF

chmod +x scripts/ralph-monitor.sh
./scripts/ralph-monitor.sh
```
