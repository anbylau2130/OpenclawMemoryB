## 一、插件概述

### 1.1 什么是插件

插件是苍穹平台扩展业务逻辑的核心机制，通过插件可以实现：
- 业务逻辑扩展

## 二、插件开发基础

### 2.1 插件基类

所有插件都继承自对应的基类：

| 插件类型 | 基类 | 说明 |
|---------|------|------|
| 动态表单插件 | AbstractFormPlugin | 所有表单插件的基类 |
| 单据界面插件 | AbstractBillPlugIn | 单据页面专用 |
| 列表插件 | AbstractListPlugin | 列表页面专用 |
| 操作插件 | AbstractOperationServicePlugIn | 操作服务专用 |
| 报表取数插件 | AbstractReportListDataPlugin | 报表取数专用 |

### 2.2 插件结构

```java
package kd.bos.plugin.demo;

import kd.bos.form.plugin.AbstractFormPlugin;
import kd.bos.form.plugin.events.*;

public class MyFormPlugin extends AbstractFormPlugin {
    
    @Override
    public void initialize() {
        // 插件初始化
        super.initialize();
    }
    
    @Override
    public void afterBindData(AfterBindDataEvent e) {
        // 数据绑定后处理
        super.afterBindData(e);
    }
    
    @Override
    public void click(EventObject evt) {
        // 按钮点击事件
        String key = evt.getSource().getKey();
        if ("btn_submit".equals(key)) {
            handleSubmit();
        }
    }
    
    private void handleSubmit() {
        // 提交处理逻辑
    }
}
```

## 三、动态表单插件

### 3.1 常用事件

| 事件 | 触发时机 | 用途 |
|-----|---------|------|
| initialize | 插件初始化 | 初始化设置 |
| afterBindData | 数据绑定后 | 数据处理后 |
| click | 按钮点击 | 处理按钮操作 |
| beforeDoOperation | 操作执行前 | 操作前校验 |
| afterDoOperation | 操作执行后 | 操作后处理 |
| propertyChanged | 属性变更 | 字段值变更处理 |
| beforeClosed | 页面关闭前 | 关闭前确认 |

### 3.2 事件处理示例

```java
@Override
public void propertyChanged(PropertyChangedEventArgs e) {
    String fieldName = e.getProperty().getName();
    
    if ("material".equals(fieldName)) {
        // 物料变更时，自动填充单价
        DynamicObject material = (DynamicObject) this.getModel().getValue("material");
        if (material != null) {
            BigDecimal price = material.getBigDecimal("price");
            this.getModel().setValue("price", price);
            
            // 计算金额
            calculateAmount();
        }
    }
}

private void calculateAmount() {
    BigDecimal qty = (BigDecimal) this.getModel().getValue("qty");
    BigDecimal price = (BigDecimal) this.getModel().getValue("price");
    
    if (qty != null && price != null) {
        BigDecimal amount = qty.multiply(price);
        this.getModel().setValue("amount", amount);
    }
}
```

## 五、操作插件

### 5.1 操作插件事件

| 事件 | 触发时机 | 用途 |
|-----|---------|------|
| onPreparePropertys | 准备属性 | 加载额外属性 |
| onAddValidators | 添加校验器 | 自定义校验 |
| beforeExecuteOperationTransaction | 操作事务前 | 事务前处理 |
| beginOperationTransaction | 事务开始 | 事务内处理 |
| endOperationTransaction | 事务结束 | 事务后处理 |
| afterExecuteOperationTransaction | 操作完成后 | 完成后处理 |

### 5.2 操作校验示例

```java
@Override
public void onAddValidators(AddValidatorsEventArgs e) {
    // 添加自定义校验器
    e.addValidator(new AbstractValidator() {
        @Override
        public void validate() {
            DynamicObject data = this.getModel().getDataEntity();
            DynamicObjectCollection entry = data.getDynamicObjectCollection("entry");
            
            if (entry == null || entry.isEmpty()) {
                this.addErrorMessage("分录不能为空");
            }
            
            // 检查明细金额
            BigDecimal totalAmount = BigDecimal.ZERO;
            for (DynamicObject row : entry) {
                BigDecimal amount = row.getBigDecimal("amount");
                totalAmount = totalAmount.add(amount);
            }
            
            BigDecimal headerAmount = data.getBigDecimal("amount");
            if (totalAmount.compareTo(headerAmount) != 0) {
                this.addErrorMessage("明细金额合计与单据金额不一致");
            }
        }
    });
}
```

| hasTrueCondition | 条件判断 | 自定义流转条件 |
| formatFlowRecord | 格式化记录 | 自定义审批记录 |
| notify | 通知 | 自定义通知逻辑 |

### 6.2 动态审批人示例

```java
@Override
public void calcUserIds(CalcUserIdsEvent e) {
    // 获取单据数据
    DynamicObject data = this.getModel().getDataEntity();
    
    // 根据金额确定审批人
    BigDecimal amount = data.getBigDecimal("amount");
    
    if (amount.compareTo(new BigDecimal("10000")) > 0) {
        // 金额超过1万，需要财务总监审批
        e.getUserIds().add("finance_director_id");
    } else {
        // 普通审批
        e.getUserIds().add("manager_id");
    }
}
```

@Override
public void afterCreateTarget(AfterCreateTargetEventArgs e) {
    // 获取目标单据

    // 获取源单据

## 八、插件开发最佳实践

### 8.1 开发规范

1. **命名规范**
   - 插件类名以`Plugin`结尾
   - 方法名使用驼峰命名
   - 常量全大写下划线分隔

2. **代码规范**
   - 添加必要的注释
   - 异常处理完善
   - 日志记录规范

3. **性能规范**
   - 避免频繁数据库操作
   - 合理使用缓存
   - 批量处理数据

### 8.2 调试技巧

```java
// 使用日志调试
Logger logger = LoggerFactory.getLogger(MyPlugin.class);
logger.info("调试信息: {}", value);

// 使用消息调试
this.getView().showMessage("调试: " + value);
```
