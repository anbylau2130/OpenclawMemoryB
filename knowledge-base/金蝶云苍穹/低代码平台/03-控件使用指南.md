## 一、控件分类

苍穹平台控件分为以下几大类：

```
控件库
├── 通用字段（基础输入控件）
├── 业务字段（业务专用控件）
├── 单据体（分录容器）
├── 通用控件（按钮、标签等）
├── 容器控件（布局容器）
├── 导航控件（菜单、工具栏）
├── 表格控件（列表、报表）
└── 图表控件（可视化图表）
```

## 二、通用字段

### 2.1 文本字段

用于输入文本内容。

| 属性 | 说明 | 默认值 |
|-----|------|-------|
| 最大长度 | 最大字符数 | 200 |
| 必填 | 是否必填 | 否 |
| 只读 | 是否只读 | 否 |
| 默认值 | 默认文本 | 空 |
| 正则校验 | 输入校验规则 | 无 |

```javascript
// 获取文本字段值
let value = this.getModel().getValue("textfield");

// 设置文本字段值
this.getModel().setValue("textfield", "新值");
```

### 2.2 数值字段

用于输入数值，包括整数、小数、长整数。

| 类型 | 说明 | 范围 |
|-----|------|------|
| 整数字段 | 整数输入 | -2^31 ~ 2^31-1 |
| 小数字段 | 小数输入 | 精度可配置 |
| 长整数字段 | 长整数输入 | -2^63 ~ 2^63-1 |

| 属性 | 说明 | 示例 |
|-----|------|------|
| 精度 | 小数位数 | 2 |
| 最小值 | 最小值限制 | 0 |
| 最大值 | 最大值限制 | 1000000 |
| 千分位 | 是否显示千分位 | 是 |

### 2.3 日期时间字段

用于选择日期和时间。

| 字段类型 | 格式 | 示例 |
|---------|------|------|
| 日期字段 | yyyy-MM-dd | 2026-03-21 |
| 长日期字段 | yyyy-MM-dd HH:mm:ss | 2026-03-21 14:30:00 |
| 时间字段 | HH:mm:ss | 14:30:00 |
| 日期范围 | 日期区间 | 2026-03-01 ~ 2026-03-21 |
| 时间范围 | 时间区间 | 09:00:00 ~ 18:00:00 |

### 2.4 选择类字段

| 字段类型 | 说明 | 使用场景 |
|---------|------|---------|
| 复选框 | 是/否选择 | 开关选项 |
| 单选 | 单选按钮组 | 互斥选项 |
| 多选 | 多选框组 | 多选场景 |
| 下拉列表 | 下拉选择 | 选项较多时 |
| 多选下拉 | 多选下拉 | 多选且选项多 |

### 2.5 其他字段

## 四、单据体

### 4.1 单据体类型

| 类型 | 说明 | 使用场景 |
|-----|------|---------|
| 单据体 | 标准分录 | 订单明细、入库明细 |
| 子单据体 | 子分录 | 复杂的主从结构 |
| 树形单据体 | 树形分录 | 层级数据 |
| 卡片分录 | 卡片式展示 | 非表格展示 |

### 4.2 单据体属性

```yaml
基础属性:
  名称: 单据体名称
  编码: 单据体编码
  数据源: 关联的数据表

显示属性:
  列配置: 显示列及顺序
  行高: 行高设置
  是否显示行号: 是/否
  是否显示合计: 是/否

编辑属性:
  是否可新增: 是/否
  是否可删除: 是/否
  是否可复制: 是/否
```

### 4.3 单据体操作

```javascript
// 获取单据体数据
let entryData = this.getModel().getValue("entryentity");

// 新增行
this.getModel().addRow("entryentity");

// 删除行
this.getModel().deleteRow("entryentity", rowIndex);

// 获取选中行
let selectedRows = this.getModel().getSelectedRows("entryentity");

// 设置行数据
this.getModel().setRowValue("entryentity", rowIndex, {
    material: { id: "xxx", name: "物料名称" },
    qty: 10,
    price: 100,
    amount: 1000
});
```

## 九、控件开发最佳实践

### 9.1 控件选择原则

1. **匹配业务场景**: 选择最合适的控件类型
2. **用户体验**: 考虑用户操作习惯
3. **性能优化**: 避免过度使用复杂控件
4. **可维护性**: 保持控件配置的一致性

### 9.2 常见问题

**Q: 如何动态隐藏/显示控件？**
```javascript
this.getView().setVisible("fieldname", false); // 隐藏
this.getView().setVisible("fieldname", true);  // 显示
```

**Q: 如何设置控件只读？**
```javascript
this.getView().setReadOnly("fieldname", true);
```

**Q: 如何动态修改控件属性？**
```javascript
this.getView().setProperty("fieldname", "property", value);
```
