## 一、页面类型概览

苍穹低代码平台支持以下PC端页面类型：

| 页面类型 | 说明 | 适用场景 |
|---------|------|---------|
| 单据 | 业务单据页面 | 订单、申请单、出入库单等 |
| 列表 | 数据列表页面 | 单据查询、数据浏览 |
| 基础资料 | 基础数据维护 | 物料、客户、供应商等 |
| 报表 | 数据报表展示 | 统计报表、分析报表 |
| 动态表单 | 通用表单页面 | 参数配置、自定义表单 |
| 参数 | 系统参数页面 | 系统设置、业务参数 |
| 卡片 | 卡片式展示 | 首页看板、数据概览 |
| 布局 | 页面布局容器 | 复杂页面布局 |

## 二、单据与列表

### 2.1 单据介绍

单据是苍穹平台最核心的页面类型，用于承载业务数据的录入、修改、查看和审批。

#### 单据核心特性
- **表头表体结构**: 支持主从表数据模型
- **多分录支持**: 支持多个子单据体
- **工作流集成**: 内置审批流程支持
- **操作配置**: 灵活的操作按钮配置
- **权限控制**: 细粒度的字段级权限

#### 单据页面属性
```properties
# 基础属性
名称: 单据名称
编码: 单据编码
描述: 单据描述

# 数据属性
主表: 关联的数据表
主键: 主键字段
编码规则: 单据编码生成规则

# 界面属性
布局方式: 标准布局/自定义布局
表头样式: 标签式/卡片式
分录样式: 表格式/卡片式
```

### 2.2 列表介绍

列表页面用于展示多条数据记录，支持查询、过滤、排序、分页等功能。

#### 列表核心特性
- **通用过滤**: 灵活的过滤条件配置
- **列配置**: 可配置显示列、列顺序、列宽
- **批量操作**: 支持批量选择、批量处理
- **数据导出**: 支持导出Excel
- **联查功能**: 支持关联查询

#### 列表属性配置
```properties
# 基础属性
名称: 列表名称
编码: 列表编码

# 数据属性
数据源: 关联的单据/基础资料
默认排序: 默认排序字段

# 显示属性
显示列: 配置显示的列
列宽: 各列宽度
是否分页: 是否启用分页
每页条数: 每页显示记录数
```

## 三、基础资料

### 3.1 基础资料介绍

基础资料用于管理企业的基础主数据，如物料、客户、供应商、员工等。

#### 基础资料特性
- **唯一性校验**: 支持配置唯一性校验规则
- **编码规则**: 自动生成编码
- **状态管理**: 启用/禁用状态
- **层级支持**: 支持树形结构
- **权限继承**: 支持数据权限控制

### 3.2 基础资料类型

| 类型 | 说明 | 示例 |
|-----|------|------|
| 普通基础资料 | 扁平结构数据 | 客户、供应商 |
| 树形基础资料 | 层级结构数据 | 组织、部门 |
| 多类别基础资料 | 多维度分类 | 物料（按类别、品牌等分类） |

## 四、报表

### 4.1 报表介绍

报表用于数据统计分析和可视化展示。

#### 报表类型
- **简单报表**: 基础的数据列表报表
- **分组报表**: 按维度分组汇总
- **交叉报表**: 多维度交叉分析
- **图表报表**: 可视化图表展示

#### 报表核心功能
- **查询条件**: 灵活的参数配置
- **数据钻取**: 支持多级钻取
- **图表联动**: 图表与数据联动
- **导出打印**: 支持导出和打印

---

## 五、动态表单

### 5.1 动态表单介绍

动态表单是通用的表单页面，适用于非标准业务场景。

#### 适用场景
- 参数配置页面
- 向导式操作页面
- 自定义查询条件
- 临时数据录入

#### 动态表单特性
- **灵活布局**: 自由布局控件
- **动态控件**: 运行时动态创建控件
- **事件驱动**: 完善的事件机制
- **数据绑定**: 灵活的数据绑定方式

## 六、页面属性配置

### 6.1 公共属性

所有页面类型共享的基础属性：

```yaml
基础信息:
  - 名称
  - 编码
  - 描述
  - 所属应用

显示设置:
  - 页面标题
  - 页面图标
  - 打开方式: 弹窗/新页签/当前页
  - 页面尺寸

权限设置:
  - 访问权限
  - 数据权限
  - 字段权限
```

### 6.2 页面事件

页面生命周期事件：

| 事件 | 触发时机 | 用途 |
|-----|---------|------|
| onLoad | 页面加载 | 初始化数据 |
| onShow | 页面显示 | 刷新数据 |
| onHide | 页面隐藏 | 保存临时数据 |
| onClose | 页面关闭 | 清理资源 |
| onAction | 操作触发 | 处理业务逻辑 |

## 七、最佳实践

### 7.1 页面设计原则

1. **用户体验优先**: 简洁直观的界面设计
2. **数据驱动**: 基于数据模型设计页面
3. **复用性**: 提取公共组件和模板
4. **性能优化**: 合理使用分页和懒加载

### 7.2 常见问题

**Q: 如何设置字段必填？**
```
在字段属性中设置"必填"属性为"是"
```

**Q: 如何实现字段联动？**
```
通过配置值更新事件，触发相关字段的数据更新
```

**Q: 如何自定义页面样式？**
```
通过CSS注入或自定义控件实现个性化样式
```
