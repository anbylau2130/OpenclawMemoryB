## 一、创建第一个低代码应用

### 1.1 前置条件

- 已安装金蝶云·苍穹开发者工具
- 已配置开发环境
- 已获得开发者权限

### 1.2 创建步骤

#### 步骤1：创建应用
1. 登录苍穹开发平台
2. 进入【云管理】→【应用管理】
3. 点击【新建应用】
4. 填写应用基本信息：
   - 应用名称
   - 应用编码
   - 应用描述
   - 所属云

#### 步骤2：创建单据
1. 进入应用设计器
2. 选择【单据】→【新建单据】
3. 配置单据属性：
   - 单据名称
   - 单据编码
   - 单据类型

#### 步骤3：设计页面
1. 使用可视化设计器
2. 拖拽控件到设计区域
3. 配置控件属性
4. 设置数据绑定

#### 步骤4：配置业务规则
1. 设置字段校验规则
2. 配置值更新事件
3. 定义业务逻辑

#### 步骤5：发布应用
1. 保存设计
2. 编译应用
3. 发布到目标环境
4. 授权用户访问

### 1.3 最佳实践

- **命名规范**：遵循统一的命名规范，便于维护
- **模块化设计**：将复杂功能拆分为独立模块
- **复用组件**：充分利用平台提供的标准组件
- **版本管理**：使用Git进行代码版本控制

## 二、苍穹开发全旅程

### 2.1 开发流程概览

```
需求分析 → 模型设计 → 页面开发 → 插件开发 → 测试验证 → 部署发布 → 运维监控
```

### 2.2 各阶段详解

#### 需求分析阶段
- 业务需求调研
- 功能模块划分
- 数据模型设计
- 界面原型设计

#### 模型设计阶段
- 单据建模
- 基础资料设计
- 工作流设计
- 权限模型设计

#### 页面开发阶段
- PC端页面开发
- 移动端页面开发
- 报表设计
- 仪表板设计

#### 插件开发阶段
- 表单插件开发
- 操作插件开发
- 工作流插件开发
- 转换插件开发

#### 测试验证阶段
- 单元测试
- 集成测试
- 性能测试
- 用户验收测试

#### 部署发布阶段
- 打包构建
- 环境部署
- 数据迁移
- 上线验证

#### 运维监控阶段
- 系统监控
- 日志分析
- 性能优化
- 问题排查

### 2.3 开发工具

| 工具名称 | 用途 | 说明 |
|---------|------|------|
| 开发者工具 | 本地开发IDE | 支持代码编写、调试、部署 |
| 设计器 | 可视化页面设计 | 拖拽式控件编排 |
| 开发助手 | 辅助开发工具 | 代码规范扫描、项目创建 |
| Monitor | 运维监控 | 系统监控、日志查看 |

### 2.4 技术栈

#### 前端技术
- **框架**: React / Vue.js
- **UI组件**: K3Cloud UI Components
- **状态管理**: Redux / Vuex
- **图表**: ECharts

#### 后端技术
- **语言**: Java 11+
- **框架**: Spring Boot
- **数据库**: PostgreSQL / MySQL
- **缓存**: Redis

#### 低代码特性
- **可视化设计**: 拖拽式页面设计
- **模型驱动**: 元数据驱动的业务建模
- **插件扩展**: 灵活的插件开发机制
- **多端适配**: PC端、移动端一体化开发
