### 1. 支持的数据库

| 数据库类型 | 版本要求 | 推荐场景 |
|-----------|---------|---------|
| PostgreSQL | 12+ | **推荐**，功能强大，性能优秀 |
| MySQL | 8.0+ | 广泛使用，生态完善 |
| Oracle | 19c+ | 大型企业，已有Oracle环境 |
| SQL Server | 2019+ | Windows环境，微软生态 |
| 达梦 | DM8+ | 国产化需求 |

### 2. 数据库对比

```yaml
PostgreSQL:
  优势:
    - ACID完整支持
    - 复杂查询性能优秀
    - JSON支持完善

  劣势:
    - 学习曲线较陡

  优势:
    - 简单易用

    - 复杂查询性能一般
    - 事务隔离级别实现不完整

  优势:
    - 功能最完整
    - 性能最优秀
    - 企业级支持
  劣势:
    - 成本高
    - 许可复杂
```

## 🐘 PostgreSQL配置

### 1. 安装PostgreSQL

**CentOS/RHEL安装**:
```bash
# 添加PostgreSQL仓库
yum install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# 安装PostgreSQL 15
yum install -y postgresql15-server postgresql15-contrib

# 初始化数据库
/usr/pgsql-15/bin/postgresql-15-setup initdb

# 启动服务
systemctl enable postgresql-15
systemctl start postgresql-15

# 验证安装
psql --version
```

**Ubuntu/Debian安装**:
```bash
# 添加PostgreSQL仓库
sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# 导入签名密钥
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | apt-key add -

apt update
apt install -y postgresql-15 postgresql-contrib-15

# 启动服务
systemctl enable postgresql
systemctl start postgresql
```



### 5. PostgreSQL备份恢复

**pg_dump逻辑备份**:
```bash
# 完整备份
pg_dump -h localhost -U postgres -d k3cloud_db -F c -f /backup/k3cloud_$(date +%Y%m%d).dump

# 仅结构
pg_dump -h localhost -U postgres -d k3cloud_db -s -f /backup/k3cloud_schema.sql

# 仅数据
pg_dump -h localhost -U postgres -d k3cloud_db -a -f /backup/k3cloud_data.sql

# 压缩备份
pg_dump -h localhost -U postgres -d k3cloud_db -F c -Z 9 -f /backup/k3cloud_$(date +%Y%m%d).dump.gz
```

**恢复数据**:
```bash
# 从自定义格式恢复
pg_restore -h localhost -U postgres -d k3cloud_db -v /backup/k3cloud_20260320.dump

# 从SQL文件恢复
psql -h localhost -U postgres -d k3cloud_db -f /backup/k3cloud_20260320.sql
```

**WAL连续归档（PITR）**:
```bash
# postgresql.conf配置
wal_level = replica
archive_mode = on
archive_command = 'rsync -a %p /backup/wal/%f'

# 基础备份
pg_basebackup -h localhost -U postgres -D /backup/base -Ft -z -P

# 时间点恢复
# 1. 停止数据库
systemctl stop postgresql-15

# 2. 清空数据目录
rm -rf /var/lib/pgsql/15/data/*

# 3. 恢复基础备份
tar -xzf /backup/base/base.tar.gz -C /var/lib/pgsql/15/data/

# 4. 创建恢复配置
cat > /var/lib/pgsql/15/data/postgresql.auto.conf <<EOF
restore_command = 'cp /backup/wal/%f %p'
recovery_target_time = '2026-03-20 10:00:00'
recovery_target_action = 'promote'
EOF

# 5. 创建恢复信号文件
touch /var/lib/pgsql/15/data/recovery.signal

# 6. 启动数据库
systemctl start postgresql-15
```

**自动备份脚本**:
```bash
#!/bin/bash
# PostgreSQL自动备份脚本

# 配置
DB_HOST="localhost"
DB_USER="postgres"
DB_NAME="k3cloud_db"
BACKUP_DIR="/backup/postgresql"
RETENTION_DAYS=30

# 创建备份目录
mkdir -p $BACKUP_DIR/{daily,weekly,monthly,wal}

# 日期
DATE=$(date +%Y%m%d)
DAY_OF_WEEK=$(date +%u)
DAY_OF_MONTH=$(date +%d)

# 每日备份
pg_dump -h $DB_HOST -U $DB_USER -d $DB_NAME -F c -f "$BACKUP_DIR/daily/k3cloud_$DATE.dump"

# 每周备份（周日）
if [ "$DAY_OF_WEEK" -eq 7 ]; then
    cp "$BACKUP_DIR/daily/k3cloud_$DATE.dump" "$BACKUP_DIR/weekly/"
fi

# 每月备份（1号）
if [ "$DAY_OF_MONTH" -eq 01 ]; then
    cp "$BACKUP_DIR/daily/k3cloud_$DATE.dump" "$BACKUP_DIR/monthly/"
fi

# 清理旧备份
find $BACKUP_DIR/daily -name "*.dump" -mtime +$RETENTION_DAYS -delete
find $BACKUP_DIR/weekly -name "*.dump" -mtime +90 -delete

# 日志
echo "$(date) - Backup completed: k3cloud_$DATE.dump" >> $BACKUP_DIR/backup.log
```

## 📊 数据库监控

### 1. PostgreSQL监控

**性能视图**:
```sql
-- 活动连接
SELECT pid, usename, application_name, client_addr, state, query, query_start
FROM pg_stat_activity
WHERE state = 'active';

-- 锁等待
SELECT blocked_locks.pid AS blocked_pid,
       blocked_activity.usename AS blocked_user,
       blocking_locks.pid AS blocking_pid,
       blocking_activity.usename AS blocking_user,
       blocked_activity.query AS blocked_statement,
       blocking_activity.query AS blocking_statement
FROM pg_catalog.pg_locks blocked_locks
JOIN pg_catalog.pg_stat_activity blocked_activity ON blocked_activity.pid = blocked_locks.pid
JOIN pg_catalog.pg_locks blocking_locks ON blocking_locks.locktype = blocked_locks.locktype
    AND blocking_locks.database IS NOT DISTINCT FROM blocked_locks.database
    AND blocking_locks.relation IS NOT DISTINCT FROM blocked_locks.relation
    AND blocking_locks.page IS NOT DISTINCT FROM blocked_locks.page
    AND blocking_locks.tuple IS NOT DISTINCT FROM blocked_locks.tuple
    AND blocking_locks.pid != blocked_locks.pid
JOIN pg_catalog.pg_stat_activity blocking_activity ON blocking_activity.pid = blocking_locks.pid
WHERE NOT blocked_locks.granted;

-- 表统计信息
SELECT schemaname, relname, n_live_tup, n_dead_tup, 
       n_dead_tup * 100.0 / NULLIF(n_live_tup + n_dead_tup, 0) AS dead_ratio
FROM pg_stat_user_tables
ORDER BY n_dead_tup DESC;

-- 慢查询
SELECT query, calls, total_time/calls AS avg_time, rows, 100.0 * shared_blks_hit /
       NULLIF(shared_blks_hit + shared_blks_read, 0) AS hit_percent
FROM pg_stat_statements
ORDER BY avg_time DESC
LIMIT 20;
```

### 2. 性能优化SQL

**索引分析**:
```sql
-- PostgreSQL
-- 查找缺失索引
SELECT schemaname, relname, attname, n_distinct, correlation
FROM pg_stats
WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
AND n_distinct > 100
ORDER BY n_distinct DESC;

-- 查找未使用的索引
SELECT schemaname, relname, indexrelname, idx_scan, idx_tup_read, idx_tup_fetch
FROM pg_stat_user_indexes
WHERE idx_scan = 0
ORDER BY pg_relation_size(indexrelid) DESC;

-- MySQL
-- 查找冗余索引
SELECT * FROM sys.schema_redundant_indexes;

-- 查找未使用的索引
SELECT * FROM sys.schema_unused_indexes;
```
