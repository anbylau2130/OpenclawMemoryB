## ⚠️ 发现的问题

### 问题1: 僵尸进程过多（29个）
```
原因: openclaw-gateway (PID 16) 的子进程未正确回收
影响: 占用进程表，不影响CPU/内存
解决: 需要重启gateway服务（不推荐，影响服务）
```

### 问题2: office-push脚本设计缺陷
```
问题: 每个agent独立运行推送脚本
- 12个进程同时运行
- 每个占用26MB内存
- 每15-30秒推送一次
- 目标相同：海辛办公室

优化方案:
1. 合并为单一推送脚本
2. 增加推送间隔（15秒→60秒）
3. 按需启动（只推送活跃agent）
```
