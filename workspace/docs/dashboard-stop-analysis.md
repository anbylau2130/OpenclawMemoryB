## 🔍 停止原因

### 直接原因
**端口冲突（EADDRINUSE）**
- 多个Dashboard进程同时尝试占用18795端口
- 导致进程崩溃并反复重启

### 日志证据
```

[FATAL] Uncaught exception — process will exit
[SHUTDOWN] SIGINT received, shutting down gracefully
```

## 🎯 可能的触发因素

### 1. 多次启动冲突
- PM2可能在短时间内多次尝试启动Dashboard
- 每次启动时前一个进程尚未完全释放端口
- 导致新进程无法绑定端口

### 2. 系统资源问题
- 内存使用46%（正常）
- CPU负载正常
- 不太可能是资源耗尽导致

### 3. 配置问题
- Dashboard监听`0.0.0.0:18795`（所有网卡）
- 可能与Gateway的18789端口配置混淆
