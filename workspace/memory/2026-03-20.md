## 复盘系统修复（16:44-16:48）

### 问题发现
复盘任务执行成功，但未生成报告文件

### 根本原因分析

**问题1：选股文件读取失败**
- `stock_review_v2.py` 的 `load_selection()` 函数只查找 `selection_{date}.json`
- 实际选股文件名为 `selection_{date}_v5_real.json`
- 导致无法加载选股数据

**问题2：复盘文件保存路径错误**
- `save_review()` 函数使用相对路径
- 保存位置：`trading-strategies/projects/stock-tracking/reviews/`
- 预期位置：`projects/stock-tracking/reviews/`

**问题3：复盘完成检查逻辑不完整**
- `scheduled_tasks.py` 只检查 `daily_review_{date}.json`
- 未检查 `weekly_review_{date}.json`（周五生成）

### 修复措施

**修复1：选股文件读取兼容**
```python
# 修改前
filename = os.path.join(selection_dir, f"selection_{today}.json")

# 修改后
possible_files = [
    os.path.join(selection_dir, f"selection_{today}_v5_real.json"),  # V5真实数据版
    os.path.join(selection_dir, f"selection_{today}.json"),  # 标准格式
    os.path.join(selection_dir, f"selection_{today}_v5.json"),  # V5版本
]

for filename in possible_files:
    if os.path.exists(filename):
        with open(filename, 'r', encoding='utf-8') as f:
            return json.load(f)
```

**修复2：复盘文件保存路径**
```python
# 修改前
output_dir = os.path.join(os.path.dirname(__file__), "..", "projects", "stock-tracking", "reviews")

# 修改后
output_dir = "/root/.openclaw/workspace/projects/stock-tracking/reviews"
```

**修复3：复盘完成检查逻辑**
```python
# 修改前
review_file = REVIEW_DIR / f"daily_review_{today}.json"
return review_file.exists()

# 修改后
review_files = [
    REVIEW_DIR / f"daily_review_{today}.json",
    REVIEW_DIR / f"weekly_review_{today}.json",
]
return any(f.exists() for f in review_files)
```

### 修复结果
- ✅ 复盘文件成功保存
- ✅ 生成每日复盘报告（`daily_review_2026-03-20.json`）
- ✅ 生成周趋势复盘报告（`weekly_review_2026-03-20.json`）
- ✅ 钉钉通知已发送
